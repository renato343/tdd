TDD -

Test Driven Development

TLDR

  • Write a test
  • Write some code
  • Run tests
  • Refactor code
  • Repeat

Why

  • Code coverage
  • Code quality
  • Code documentation
  • Code confidence

Red & Green Button

  • Red - Test Fails
  • Refactor - Clean up code
  • Green - Test Passes
  • Repeat - Fearless Programming

RULES

  • Not Allowed to write Production Code until you have a failing test because production code doesn't exist
  • Not Allowed to write more of a test that is suficent to fail (COMPILING IS FAILING)
  • Not Allowed to write production code more than enough to pass the test
  • Refactor the code
  • Repeat
